Oracle中maven pom jdbc的url的 单机写法 和 双机写法

介绍 SID ServerName的区别
以及 RAC 方式

  • 普通SID方式
  • 普通ServerName方式
  • RAC方式 双击写法

普通SID方式

SID是实例名，实例名指的是用于响应某个数据库操作的数据库管理系统的名称。实例名是由初始化参数文件的参数instance_name决定的。如果这个参数不被指定(即instance_name没有被指定为任何值)，那么实例的名字由该用户的环境变量ORACLE_SID（注意这里是大写）决定

jdbc:oracle:thin:username/password@x.x.x.1:1521:SID

普通ServerName方式

SERVICE_NAME指的是listener中的全局数据库名：这个名字是由listener.ora中GLOBAL_DBNAME参数决定的。这个名字代表的是客户端连接到数据库时，tnsnames.ora中SERVICE_NAME参数所对应的值

jdbc:Oracle:thin:username/password@//x.x.x.1:1522/ABCD

RAC方式 双击写法

jdbc:oracle:thin:@(DESCRIPTION=
(ADDRESS_LIST=(ADDRESS=(PROTOCOL=TCP)(HOST=x.x.x.1)(PORT=1521))
(ADDRESS=(PROTOCOL=TCP)(HOST=x.x.x.2)(PORT=1521)))
(LOAD_BALANCE=yes)(CONNECT_DATA=(SERVER=DEDICATED)(SERVICE_NAME=xxrac)))

RAC全称real application clusters，译为“实时应用集群”，即两台主机采取双机互备模式，共同采用相同的共享存储，能够实现用户在两台机器上访问的负载均衡。当一台主机出现问题（DOWN机等）时，后台应用会被透明切换至另一台主机，这种机制对于一些7*24小时的业务比较常用到。

1. 项目中的单机模式(pom.xml)

<jdbc.url>
jdbc:oracle:thin:@(DESCRIPTION=(ADDRESS=(PROTOCOL=TCP
(HOST=192.168.121.189)(PORT=1521))(ADDRESS=(PROTOCOL=TCP
(HOST=192.168.121.191)(PORT=1521))(LOAD_BALANCE=no
(CONNECT_DATA=(SERVER=DEDICATED)(SERVICE_NAME=d4cwf
(FAILOVER_MODE=(TYPE=SELECT)(METHOD=BASIC)(RETRIES=180)(DELAY=5))))
</jdbc.url>                

2. 项目中的双机模式(pom.xml)

<jdbc.url>
jdbc:oracle:thin:@(DESCRIPTION=(ADDRESS=(PROTOCOL=TCP)(HOST=192.168.121.189
(PORT=1521))(ADDRESS=(PROTOCOL=TCP)(HOST=192.168.121.191)(PORT=1521)
(LOAD_BALANCE=no)(CONNECT_DATA=(SERVER=DEDICATED)(SERVICE_NAME=d4cwf
(FAILOVER_MODE=(TYPE=SELECT)(METHOD=BASIC)(RETRIES=180)(DELAY=5))))
</jdbc.url>
